# Staging Documentation Directory

This directory contains documentation files created during development that are not yet integrated into the main documentation structure.

## Current Files

### developer_notes.md
Personal workspace for tracking thoughts, tools, and ideas that don't directly influence the project but are worth keeping for future reference. This includes:
- Tool evaluations and considerations
- Architecture ideas and concepts
- Performance notes and benchmarks
- Security considerations
- Meeting notes and business logic insights

### frontend_issues.md
Issue tracker for frontend problems discovered during usage that need systematic addressing:
- UI/UX issues (chat positioning, RTL layout problems)
- Functionality issues
- Performance and accessibility concerns
- Browser compatibility issues

## Usage Guidelines

- Create new documentation files here during development
- Use descriptive filenames that clearly indicate the feature or concept
- Follow the reference-based approach (no code snippets, file references only)
- Files will be periodically reviewed and integrated into the main documentation structure
- These files serve as the context hub for coding assistants to understand ongoing work and decisions
